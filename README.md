# Hash Table Fundamentals and Implementation – Practice Problems

## Project Overview

This project demonstrates the implementation and real-world applications of **Hash Tables** using Java.
It includes multiple practice problems that show how hash tables are used to achieve **fast data lookup, efficient storage, and high-performance systems**.

Hash tables provide **average O(1) time complexity** for insertion, deletion, and search operations, making them one of the most important data structures in modern software systems.

---

## Concepts Covered

The following hash table concepts are implemented in this project:

* Key–Value Mapping
* Hash Functions
* Collision Handling
* Load Factor
* Chaining
* Open Addressing
* Frequency Counting
* Real-Time Data Processing
* Time and Space Complexity Analysis

---

## Practice Problems Implemented

### 1. Social Media Username Availability Checker

A system that checks whether a username is available using a hash table and suggests alternatives if the username is already taken.

### 2. E-commerce Flash Sale Inventory Manager

Manages product inventory during high-traffic flash sales and prevents overselling using efficient hash-based stock tracking.

### 3. DNS Cache with TTL

A DNS caching system that stores domain-to-IP mappings with expiration time to reduce lookup latency.

### 4. Plagiarism Detection System

Detects similarity between documents by storing and comparing n-grams using hash tables.

### 5. Real-Time Website Analytics Dashboard

Tracks page views, traffic sources, and unique visitors using multiple hash tables.

### 6. Distributed API Rate Limiter

Implements a token bucket rate-limiting mechanism to control API requests per client.

### 7. Search Engine Autocomplete System

Stores search queries with frequency counts and provides suggestions based on prefix matching.

### 8. Parking Lot Management System

Uses open addressing and linear probing to assign parking spots based on vehicle license plates.

### 9. Two-Sum Financial Fraud Detection

Detects transaction pairs that sum to a target amount using hash table complement lookup.

### 10. Multi-Level Cache System

Implements a caching architecture with L1, L2, and database layers to optimize data retrieval performance.
